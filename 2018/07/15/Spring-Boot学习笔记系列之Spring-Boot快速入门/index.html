Spring Boot学习笔记系列之Spring Boot快速入门

Spring Boot 学习
  • Spring Boot

本篇主要是Spring Boot的入门开发的基本知识，介绍如何使用Intellij IDEA 工具创建一个简单的hello world项目流程。

开发准备

  1. 开发环境JDK 1.8+
  2. 开发工具( Intellij IDEA )
  3. 项目管理工具（ Maven 3.3.9 ）
  4. Spring Boot 版本2.0.3.RELEASE

快速创建项目

第一步：
打开IntelliJ IDEA，在菜单栏选择File菜单–>New–>Project...–>Spring Initializr，Project SDK即选择JDK的版本，Choose Initializer Service URL选择 Default:http://start.spring.io

由于我这里用的多模快下创建项目，所以选择的是New Module,创建出来的项目都一样


第二步：
在接下来的页面我们创建一个Maven项目，如果有朋友对Maven不了解的话，可以参考网上的很多教程，这里不再详述了，我们在下图页面中填写项目相关的信息，接下来选择项目需要的依赖，我们在这里先勾选Web即可，在pom文件中变回生成Spring Boot Web相关的jar包，同时可以选择Spring Boot的版本。其他的依赖在此处不勾选也没有关系，需要的话可以在pom.xml文件中添加。


第三步：
项目创建完成后会自动生成很多文件，其中BootHelloApplication是用于启动Spring Boot项目的，有一个main方法，我们知道main方法是程序的入口，因此右键点击Run 'BootHelloApplication'来运行程序。

其中是@SpringBootApplication组合注解，兼备了@EnableAutoConfiguration和@ComponentScan 注解的功能。

IntelliJ IDEA中集成了Tomcat，因此我们不需要在开发工具中针对Tomcat做任何配置，在运行结果中我们可以看到已经启动了Tomcat的8080端口，这个是Tomcat的默认端口。

第四步：
快速访问 输入http://localhost:8080/hello
效果图如下，是不是很Easy?


@RestController的意思就是controller里面的方法都以json格式输出，不用再写什么jackjson配置的了！

Spring Boot 项目结构


如上图所示，Spring Boot的基础结构共三个文件:

  1. src/main/java 程序开发以及主程序入口，自动生成启动类项目名+Application.java

  2. src/main/resources 配置文件，自动生成了配置文件application.properties

  3. src/test/java 测试程序 自动生成测试类项目名+ApplicationTests.java

Spring Boot Pom 文件


在Spring Boot中，模块的依赖都是以starter的方式进行，以 spring-boot-starter-方式命名，指明了具体的模块。Spring Boot生态中提供了丰富的starter供开发者使用在上面的技术栈中可以看到。
这种模块化的依赖，简化了大量的依赖配置。使用starter的依赖方式，可以把相应需要依赖的包一起依赖到项目中，而不像传统的Spring 需要各自依赖包。
如在Spring Boot中web模块，在传统的Spring配置中需要依赖web服务的包,在Spring Boot中，只需要依赖spring-boot-starter-web就能把相应的包一并依赖进来，极大的简化了配置。

pom.xml中依赖 spring-boot-starter-web 模块，包括了Tomcat和spring-webmvc不需要指定version 版本，因为父模块中已经有默认配置，如果需要指定版本可添加。

pom.xml中依赖 spring-boot-starter-test 测试模块，包括JUnit、Hamcrest、Mockito ;

Spring Boot 单元测试

打开的src/test/下的测试入口，编写简单的http请求来测试；使用mockmvc进行，利用MockMvcResultHandlers.print()打印出执行结果。

Spring Boot 核心注解

Spring Boot 推荐采用基于 Java Config 的配置方式，而不是传统的 XML。

  1. @SpringBootApplication
    让Spring boot 自动给程序进行必要配置，这个配置等同于：@Configuration, @EnableAutoConfiguration 和@CompoentScan。
  2. @Configuration
    相当于传统的xml配置文件，如果有些第三方库需要用到xml文件，建议仍然通过@configuration类作为项目的配置主类–可以使用@ImportResource注解加载xml配置文件
  3. @EnableAutoConfiguration
    Spring Boot 自动配置：尝试根据你添加jar依赖自动配置你的spring应用。
  4. @CompoentScan
    表示将该类自动发现（扫描）并注册为Bean,并结合@Autowired注解导入。如果没有配置的话，SpringBoot会扫描启动类所在包下以及子包下使用了@service,@Repository等注解的类。
  5. @PropertySource
    如果需要自定义的属性文件需要加载，可以使用该注解进行注入，并且用@Value配合使用
  6. @Bean
    用@Bean标注方法等价于XML中配置的bean.
  7. @Value
    注入Spring Boot application.properties配置的属性的值。

总结

  • Spring Boot 使编码变简单
  • Spring Boot 使配置变简单
  • Spring Boot 使部署变简单
  • Spring Boot 使监控变简单
